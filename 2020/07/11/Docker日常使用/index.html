Docker日常使用

泡泡机器人SLAM与你一同成长

❤

基本操作

拉取镜像后 docker pull paopaorobot/ros-vnc
不通过UI界面进入ubuntu，可以直接运行docker run repositoryname
cat /etc/issue 可查看当前ubuntu的版本
后面该怎么操作怎么操作，使用完之后exit退出即可。

打开界面

有几种方式可以在docker中设置界面，openbox，vnc等。
这里选择的是ros-vnc来进行可视化界面的，vnc其实是一个功能， ubuntu自带的某些软件也有这个功能。
直接用docker run -it -p 5900:5900 paopaorobot/ros-vnc打开即可。

调整分辨率

docker run -it -p 5900:5900 -e RESOLUTION=1980x1200 paopaorobot/ros-vnc

通过ssh连接

docker run -it -p 2222:22 paopaorobot/ros-vnc
打开本机控制台
ssh -o 'UserKnownHostsFile=/dev/null' root@localhost -p [port] port表示端口号，保持和开启镜像时映射的端口一致。
在控制台使用下述ssh链接打开的容器：
ssh root@localhost -p 2222
也可以设置自定义的ssh密码,如asc123
docker run -it -p 2222:22 -e SSHPW=asc123 paopaorobot/ros-vnc
也可以随机选择端口号，P会系统自动给一个端口，可以避免端口冲突
docker run -it -P paopaorobot/ros-vnc
通过docker port查看当前开了哪些端口，然后找到想要的端口

同时使用ssh和可视化界面

docker run -it -p 5900:5900 -p 2222:22 paopaorobot/ros-vnc
docker run -itd -p 5900:5900 -e SSHPW=usmn9a -e RESOLUTION=1920x1280 -v /d/data:/data -p 2222:22 amazinghao/ubuntu:graph_slam_updated
-p 将本地5900端口映射到容器，服务于vnc进行可视化
-e SSHPW= 设置ssh连接的密码为xxx
-p 将本地2222端口映射到容器，服务于ssh进行连接
-e RESOLUTION=1920x1280 设置分辨率为1080p
-v 表示将本地D:/data路径挂在到容器对应的/data下,就不用将数据拷贝到容器中了。
amazinghao/ubuntu:graph_slam_updated 表示将要实例化的镜像名称。

退出之后再打开
  • 按照前述方法用vnc或者ssh打开docker的一个镜像并生成容器之后，通过exit退出当前容器，若是后续还想再进去，此时如果通过run命令就会再次产生一个容器，而你在之前的容器中所做的操作就不会保留。
  • 因此这时需要通过start，打开之前操作过现在状态是exit的容器，用attach建立连接，此时如果通过docker ps -a可以发现之前的容器又可以打开了,并且端口号是可用的。
  • 另外如果想要在exit容器时，容器不退出，应该使用exec命令打开镜像docker exec -it 24d1106f11b8 /bin/bash,此时也会发现容器时可用的了。

    制作Demo

    通过-v可以将本地数据路径挂载到docker中
    docker run -it -p 5900:5900 -v localdatadir:ubuntudir paopaorobot/ros-vnc
    roslaunch xx xxx.launch 参数（可能是数据路径，具体看不同的接口说明）

    Docker远程debug

    把代码挂在到容器中
    docker run -it -v localcodedir:ubuntucodedir paopaorobot/ros-vnc
    e.g.用vscode远程连接到容器，Remote-Container/Remote-SSH，cd到对应目录下即可
    要先把本地代码挂在到远程，然后远程连接到容器中，利用容器的环境进行开发，build，run
    本地修改之后，远程容器中也会发生响应的变化。

    出现问题以及解决

    1.vnc首次可以连接成功，但是再次连接时失败
    打开vnc5900时提示，connected closed unxepectedly
    解决方案： 这是需要使用start将上次的docker容器打开就可以了，连接失败的原因可能是因为端口被占用了。
